## 可行性分析

**文字内容**：
**技术可行性**

- 前端：Vue 3 + Element Plus + Pinia
- 后端：Spring Boot + JPA + SQLite/MySQL
- 安全：Spring Security + JWT
  **经济可行性**
- 部署成本低：支持轻量级部署
- 维护效率高：前后端分离，易于扩展

**配图内容**：

- 技术栈图标矩阵（Vue, SpringBoot, Element Plus, TypeScript）
- 成本效益对比图（传统人工收集 vs 系统自动化）

**讲解文字**：
"我们通过技术与经济两个维度验证了项目可行性。技术上采用成熟的Spring Boot与Vue3生态，确保系统稳定高效；经济上，自动化作业收集大大降低了人工管理成本，ROI回报显著。"

## 需求分析

**文字内容**：
**核心需求**：

1. **作业收集**：老师发布任务，学生上传，支持批量下载。
2. **点对点传输**：用户间私密发送文件，类似邮件体验。
3. **权限控制**：基于角色的访问控制（RBAC），区分学生、老师、管理员。
   **质量需求**：

- 响应迅速：界面交互流畅，无卡顿。
- 安全可靠：文件加密存储，操作全程审计。

**配图内容**：

- 用户角色用例图（学生、老师、管理员）
- 需求优先级金字塔

**讲解文字**：
"通过调研，我们确定了三大核心场景：作业收集、P2P文件流转以及严格的权限管理。同时，系统对数据安全性和操作流畅度提出了高标准要求。"

---

## 系统架构设计

**文字内容**：
**三层架构**：

- **表现层**：Vue 3单页应用，响应式布局，Glassmorphism风格。
- **业务层**：Spring Boot RESTful API，处理核心逻辑。
- **数据层**：JPA ORM映射，支持多种数据库后端。
  **关键技术**：
- **AES加密**：保障文件存储安全。
- **WebSocket/轮询**：实时更新消息通知。
- **Audit Log**：全链路操作审计。

**配图内容**：

- 前后端分离架构图
- 核心流程数据流向图

**讲解文字**：
"系统采用经典的前后端分离架构。前端注重交互体验，使用了最新的Vue3组合式API；后端基于Spring Security构建了坚固的安全防线，并引入了文件加密与审计日志机制。"

---

## 详细设计（核心模块）

**文字内容**：
**任务收集模块**：

- **流程**：老师创建任务 -> 设定截止时间/权限 -> 学生上传 -> 老师查阅/下载。
- **亮点**：支持设置“学生是否可互看”，满足不同教学场景。
  **点对点传输模块**：
- **流程**：搜索用户（学号/姓名） -> 附加备注 -> 发送文件 -> 接收方下载。

**配图内容**：

- 任务状态流转图
- P2P发送时序图

**讲解文字**：
"以任务收集模块为例，老师可以灵活控制任务的可见性与截止时间，系统会自动归档所有提交的文件。点对点模块则提供了便捷的校内文件直传通道，替代了传统的邮件或即时通讯软件传文件的方式。"
